# Internationalization (admin UI)

Admin UI strings are externalized via **@nuxtjs/i18n** v10 (vue-i18n v11). Default locale is **English**; **Russian** has full key parity.

## Configuration

| Setting | Value | Rationale |
|---------|-------|-----------|
| Module | `@nuxtjs/i18n` | Official Nuxt module; lazy locales, cookie, composables |
| `defaultLocale` | `en` | Base language for new keys |
| `strategy` | `no_prefix` | Same URLs for all locales (admin SPA, no SEO) |
| `langDir` | `i18n/locales/` | Module v10 layout (`en.json`, `ru.json`) |
| `vueI18n` | `i18n/i18n.config.ts` | Composition API only (`legacy: false`) |
| Cookie | `i18n_locale` | Persists manual switch and first detection |
| Detection | `detectBrowserLanguage` on `/` | `ru*` → `ru`, else `en` |

Document title: `layouts/default.vue` sets `titleTemplate` from `useAppName()` (`APP_NAME` env → `runtimeConfig.public.appName`, default `Telemodai` in `lib/app-config.ts`). Page segments use `usePageTitle(() => t('page.*.documentTitle'))`.

Footer **EN · RU** on the version row (`{appName} vX.Y.Z — tagline | EN RU`): `components/layout/AppFooter.vue`, `components/layout/LocaleSwitcher.vue` (`useAppLocale` → `setLocale`). Same locale switcher on `/login`.

`lib/app-config.ts` — `DEFAULT_APP_NAME`, `resolveAppName()`, `APP_LINKS.authorSite`.

## Key naming

Nested JSON, dot keys in code:

```
nav.dashboard
page.bots.title
bot.chats.addChat
common.save
footer.releaseNotes
chatActivation.completed
botTemplate.htmlHelp.sections.format.title
```

Convention: `page.*` for page chrome; `common.*` for shared actions/errors; domain prefixes (`bot`, `moderation`, `dashboard`, …).

## Patterns in code

| Case | Pattern |
|------|---------|
| Template | `{{ $t('key') }}` or `useI18n().t('key')` |
| Interpolation | `t('common.pageOf', { page, totalPages })` |
| Arrays (prerequisites) | `tm('chatActivation.prerequisites')` |
| Breadcrumbs | `usePageBreadcrumbs(() => [{ label: t('nav.bots'), to: '/bots' }, …])` |
| `confirm()` | `t('common.confirm.removeChat')` |
| Dates | `toLocaleDateString(locale.value === 'ru' ? 'ru-RU' : 'en-US', …)` |
| Placeholder chips | `labelKey` / `hintKey` in `lib/bot-message-template-ui.ts` → `t(chip.labelKey)` |

## Locale file pitfalls

vue-i18n parses `{name}` and `@` specially. In `i18n/locales/*.json`:

- **Telegram template placeholders** (`{user_mention}`, `{rule_name}`, …) → escape as `{'{'}user_mention{'}'}` when shown as literal text.
- **@username / @{botId}** → escape `@` as `{'@'}` before `{botId}` or literal usernames.
- **HTML in help copy** — allowed; `nuxt.config` sets `i18n.compilation.strictMessage: false`.

After editing locales, run `bun test tests/unit/i18n/locale-key-parity.test.ts` — `ru.json` must match `en.json` keys.

## Out of scope (not in locale files)

| Area | Reason |
|------|--------|
| Per-bot Warning/Ban message templates | User/bot content sent to Telegram chats |
| `RULE_TEMPLATES` preset catalog | Server-side rule definitions |
| LLM moderation prompts | Runtime AI |
| Release notes body (`data/releases/`) | Generated per release; separate product decision |
| API `statusMessage` strings | Server errors; future client-side error dictionary |

## Adding UI copy

1. Add key to `i18n/locales/en.json`.
2. Add same key path to `i18n/locales/ru.json` (Russian translation).
3. Replace hardcoded string in Vue/composable with `$t()` / `t()`.
4. Run `bun test` (parity + affected unit tests) and `bun run build`.
